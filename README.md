# Формула: Современная Low-Code платформа для автоматизации бизнеса

Прототип low-code платформы для быстрой разработки учетных и бизнес-приложений.

## Описание Системы

**Формула** — это гибкая платформа-конструктор, позволяющая создавать учетные системы любой сложности (CRM, WMS, ERP) без написания тысяч строк кода с нуля. Она объединяет в себе простоту использования для конечного пользователя и мощные инструменты разработки для инженеров.

В основе системы лежит **метаданные-ориентированная архитектура**: вы описываете *что* хотите получить (справочник клиентов, документ продажи), а система сама строит интерфейс, базу данных и API.

---

## 1. Три кита системы (Для Пользователя)

Вся логика Формула строится вокруг трех базовых понятий, знакомых любому, кто работал с учетными программами.

### 1. Справочники (Nouns / Существительные)
Это картотеки вашей компании. Здесь хранится условно-постоянная информация: список клиентов, номенклатура товаров, сотрудники, склады.
*   Поддерживают иерархию (папки).
*   Могут иметь любые поля: от простого текста до картинок и ссылок на другие справочники.

> **[📸 Скриншот: Форма списка Справочника "Контрагенты" или "Номенклатура"]**
> *Примечание: Показать список с колонками и папками.*

### 2. Документы (Verbs / Глаголы)
Это события, происходящие во времени. "Заказ клиента", "Приходная накладная", "Списание".
*   Всегда имеют **Номер** и **Дату**.
*   Содержат **Табличные части** (например, список товаров в накладной).
*   Имеют функцию **"Проведение"**: документ превращается из черновика в факт, изменяющий остатки.

> **[📸 Скриншот: Форма Документа "Реализация товаров" с заполненной табличной частью]**
> *Примечание: Показать шапку документа и таблицу с товарами.*

### 3. Регистры (Scoreboard / Табло)
Это место, где система "подбивает итоги". Пользователи не редактируют их вручную.
*   **Регистры накопления:** Считают остатки (сколько товара на складе) и обороты (на какую сумму продали).
*   **Регистры сведений:** Хранят историю значений (цена товара на конкретную дату, курс валют).

## 2. Архитектура и Технологии (Для Разработчика)

Формула построена на современном стеке технологий, обеспечивающем высокую производительность и безопасность.

*   **Frontend:** [Next.js](https://nextjs.org/) (React 19.2.3) + Tailwind CSS 4.1.9 + Radix UI. Интерфейс полностью динамический — он рендерится на лету на основе JSON-конфигурации (метаданных).
*   **Backend:** Next.js API Routes (Node.js).
*   **Database:** MySQL 3.15.3. Система автоматически создает и мигрирует таблицы в БД при изменении структуры объектов в конфигураторе.
*   **ORM/Query Builder:** Собственная реализация для работы с динамическими структурами данных.

### Ключевая особенность: Обфускация кода
Для защиты интеллектуальной собственности, исполняемый JS-код (в модулях страниц и API) хранится в базе данных в открытом виде (для удобства редактирования администратором), но отдается клиенту (браузеру) в **компилированном и обфусцированном виде**. Это происходит прозрачно на уровне сервера Next.js.

## 3. Инструменты Конфигуратора

Сердце системы — это режим **Конфигуратора**, где администратор строит приложение.

### Редактор Метаданных (MetaEditor)
Здесь создается структура данных. Вы добавляете поля разных типов:
*   **Примитивы:** Строка, Число, Булево (Да/Нет), Дата.
*   **Ссылки:** Связи между объектами (например, поле "Клиент" ссылается на справочник "Контрагенты").
*   **UI элементы:** Кнопки, Надписи, Картинки.

У каждого поля есть гибкие настройки: `Обязательное`, `Только чтение`, `Скрытый`, `Автозаполнение` (подтянуть адрес при выборе клиента) и `Фильтрация` (показать города только выбранной страны).

> **[📸 Скриншот: Редактор метаданных (MetaEditor), открытая боковая панель со свойствами реквизита]**
> *Примечание: Видно список полей слева и настройки (галочки Required, Hidden) справа.*

### Конструктор Форм (Layout Designer)
Визуальный Drag-and-Drop редактор. Вы "расставляете мебель" на форме: группируете поля, создаете вкладки и колонки, настраиваете ширину элементов.

> **[📸 Скриншот: Конструктор форм (FormLayoutEditor) в процессе перетаскивания поля]**

### Редактор Страниц (Page Editor)
Позволяет создавать полностью кастомные страницы (HTML/CSS/JS), выходящие за рамки стандартных форм.
*   **Интерфейс:** Вкладки для HTML, CSS, JS и Live-Preview.
*   **Безопасность:** Встроенная защита и обфускация JS кода перед отправкой клиенту.

> **[📸 Скриншот: Page Editor с новыми цветными табами (HTML/CSS/JS) и панелью кода]**

## 4. Логика и Автоматизация

Формула позволяет программировать бизнес-логику любой сложности.

### Модули и `formApi`
К каждой форме можно прикрепить скрипт на JavaScript. Система предоставляет мощный интерфейс `formApi` для взаимодействия с формой:
*   `getValue()`, `setValue()` — управление данными.
*   `read()`, `save()`, `delete()` — прямые запросы к базе данных.
*   `showAlert()`, `showConfirm()` — взаимодействие с пользователем.
*   `Почта()` — отправка email уведомлений.

**Пример:** При изменении поля "Сумма" автоматически пересчитывать "НДС" или запретить закрытие формы, если не заполнен комментарий.

### Проведение и Движения
Настройка того, как документ влияет на регистры (`postingRules`). Это делается в визуальном редакторе: вы просто соединяете стрелочками поле документа (например, "Товар") с измерением регистра.

> **[📸 Скриншот: Редактор правил проведения (PostingRulesEditor)]**

### Ввод на основании
Правила, позволяющие создать один документ из другого (например, "Акт" из "Счета") с автоматическим переносом всех данных.

## 5. Вывод данных

### Печатные формы
Генерация Pixel-perfect документов (Счета, УПД) на основе HTML-шаблонов. Поддерживаются штрих-коды, QR-коды и сложные выборки данных (`dataSelector`) через JS.

> **[📸 Скриншот: Пример готовой печатной формы (Счет на оплату)]**

### Конструктор Отчетов (Report Builder)
Инструмент для аналитики, похожий на сводные таблицы Excel.
*   **Источники:** Объединение данных из разных регистров и справочников.
*   **Настройки:** Группировки, отборы (фильтры), агрегация (сумма, количество, среднее).
*   **Визуализация:** Табличное представление с итогами.

> **[📸 Скриншот: Окно настроек отчета (вкладки Поля, Группировки) и результат формирования]**

## 6. Внешний API (Интеграция)

Система готова к работе в IT-ландшафте предприятия.
*   **REST API:** Полный доступ ко всем объектам (CRUD) по адресу `/api/external`.
*   **Auth:** JWT-авторизация.
*   **Возможности:** Чтение списков с фильтрацией, создание документов, запуск проведения, работа с печатными формами.

Это позволяет легко подключить к Формуле сайт интернет-магазина, мобильное приложение или Telegram-бота.
