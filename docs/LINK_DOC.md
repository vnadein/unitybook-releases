# Реквизит "Документ Ссылка": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем свойствам реквизита "Документ Ссылка" (тип `document_reference`). Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

Реквизит "Документ Ссылка" очень похож на "Справочник Ссылку", но предназначен для связи не со справочниками, а с другими документами в системе. Например, в документе "Оплата" можно сослаться на документ "Счет на оплату", на основании которого производится платеж.

**Как это выглядит:** Внешне поле идентично ссылке на справочник. Но при выборе значения откроется список документов указанного типа. Главное отличие — в текстовом представлении выбранного значения. Для документов оно обычно выглядит так:

`Реал. №15 от 28.11.2024`

Это представление помогает быстро идентифицировать документ по его типу, номеру и дате.

---

### Реализованные свойства

Эти свойства полностью работают и определяют поведение поля-ссылки на документ.

#### `Ссылка на` (`referenceTo`)
*   **Что делает:** **Это ключевое свойство.** Оно указывает, на какой именно тип документа будет ссылаться реквизит.
*   **Как работает:** Вы указываете системный `id` нужного документа (например, `doc_sales` для "Реализации товаров"). Когда пользователь нажмет кнопку выбора, система откроет список именно этих документов.

#### `Обязательный` (`required`)
*   **Что делает:** Делает выбор документа в этом поле обязательным.
*   **Как работает:** К названию поля добавляется красная звездочка (`*`), а пустое поле подсвечивается красной рамкой.

#### `Только чтение` (`readOnly`)
*   **Что делает:** Запрещает пользователю изменять выбранный документ.
*   **Как работает:** Поле и кнопка выбора становятся неактивными (серыми).

#### `Скрытый` (`hidden`)
*   **Что делает:** Полностью скрывает поле и его название из интерфейса формы.
*   **Как работает:** Поле не отображается, но его значение-ссылка на документ существует и может использоваться в других частях системы.

#### `Автозаполнение` (`autofillFrom`)
*   **Что делает:** Автоматически заполняет поля текущего документа на основе данных из выбранного по ссылке документа.
*   **Как работает:** Это свойство настраивается у полей-приемников (например, текстовое поле "Контрагент"). В настройках автозаполнения вы указываете поле-ссылку (например, "Счет-основание") и выбираете, какой из его реквизитов (например, "Контрагент") нужно скопировать. Когда пользователь выберет счет, связанный с ним контрагент автоматически подставится в текущий документ.

### Фильтрация по родителю (документу-основанию)

Как и в случае со справочниками, для ссылок на документы есть мощный механизм фильтрации, который позволяет показывать в списке выбора только нужные документы.

*   **Пользовательский термин:** `Родитель` или `Документ-основание`
*   **Свойство в коде:** `filter`

**Как это работает на примере "Договор -> Доп. соглашение":**

Представьте, у вас есть документ "Дополнительное соглашение", в котором нужно указать основной "Договор". Вы хотите, чтобы при выборе связанных с этим доп. соглашением счетов, в списке показывались только те счета, которые также относятся к основному "Договору".

1.  **Настройка:** Для реквизита-ссылки "Счет" в редакторе метаданных настраивается свойство `filter`. В нем указывается, что поле "Договор" в списке счетов должно быть равно значению, выбранному в поле "Договор" на текущей форме.
2.  **Работа в интерфейсе:**
    *   Пользователь создает "Доп. соглашение" и выбирает основной "Договор" (например, Договор №123).
    *   Теперь, когда он переходит к выбору "Счета", в списке он увидит только те счета, которые в своей карточке также ссылаются на Договор №123.

Этот механизм позволяет создавать строгие и логичные связи между документами, исключая возможность ошибки пользователя.
