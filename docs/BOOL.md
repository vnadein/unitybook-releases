# Реквизит "Булево": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем свойствам реквизита с типом "Булево" (логический тип). Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

Реквизит типа "Булево" представляет собой флажок (чекбокс), который может находиться в двух состояниях: "да" (галочка стоит) или "нет" (галочка снята).

---

#### `По умолчанию` (`defaultValue`)
*   **Что делает:** Задает состояние флажка, которое будет установлено по умолчанию при создании **нового** объекта.
*   **Как работает:** Вы можете указать `true` (галочка будет стоять) или `false` (галочка будет снята). При открытии формы нового объекта флажок примет это состояние.
*   **Пример:** Для поля "Активен" можно установить значение по умолчанию `true`, чтобы новые создаваемые элементы были сразу активны.

#### `Только чтение` (`readOnly`)
*   **Что делает:** Запрещает пользователю изменять состояние флажка.
*   **Как работает:** Флажок становится неактивным (серым), и на него нельзя нажать. Его значение можно изменить только автоматически (например, через `Автозаполнение`).

#### `Скрыть` (`hidden`)
*   **Что делает:** Полностью скрывает флажок и его название из интерфейса формы.
*   **Как работает:** Поле не отображается на форме, но его значение (true/false) существует и может использоваться в других частях системы.

#### `Автозаполнение` (`autofillFrom`)
*   **Что делает:** Автоматически устанавливает или снимает галочку на основе значения из другого объекта по ссылке.
*   **Как работает:** Если в форме есть поле-ссылка (например, "Тип договора"), то для флажка "Требуется согласование юриста" можно настроить автозаполнение. Когда пользователь выберет тип договора, у которого стоит признак необходимости согласования, галочка "Требуется согласование юриста" установится автоматически.

---

### Свойства с особым поведением

#### `Обязательный` (`required`)
*   **Что делает:** Помечает поле как обязательное.
*   **Как работает:**
    1.  К названию поля добавляется красная звездочка (`*`).
    2.  Если флажок **не установлен** (имеет значение `false`), поле подсвечивается красной рамкой.
*   **Важно:** С точки зрения системы, "незаполненным" для булевого поля считается состояние "нет" (`false`). Это может быть не всегда очевидно для пользователя, так как "нет" — это тоже определённое значение. Используйте это свойство, только если по логике вашего бизнес-процесса флажок **обязательно должен быть включен**.
