# Реквизит "Число": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем свойствам реквизита с типом "Число". Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

---

### Реализованные свойства

Эти свойства полностью работают. Вы можете смело использовать их при настройке объектов.

#### `Обязательный` (`required`)
*   **Что делает:** Делает поле обязательным для заполнения.
*   **Как работает:** К названию поля добавляется красная звездочка (`*`), а пустое поле подсвечивается красной рамкой. Система не позволит сохранить объект, пока поле не будет заполнено.

#### `По умолчанию` (`defaultValue`)
*   **Что делает:** Задает число, которое будет автоматически подставлено в поле при создании **нового** объекта.
*   **Как работает:** При открытии формы нового объекта это значение появляется в поле. Для уже существующих записей оно не применяется.

#### `Выражение` (`expression`)
*   **Что делает:** Позволяет автоматически вычислять значение поля на основе значений других полей.
*   **Как работает:**
    1.  В свойство `expression` записывается математическая формула. Имена других полей нужно указывать в квадратных скобках. Например: `[Количество] * [Цена]`.
    2.  Поле, у которого задано выражение, **автоматически становится "Только для чтения" (`readOnly`)**, так как его значение вычисляется, а не вводится вручную.
    3.  При изменении любого поля, от которого зависит формула, система на лету пересчитывает и обновляет значение.
    4.  Поддерживается агрегация данных из табличных частей, например: `SUM([Товары].[Сумма])`.
*   **Пример:** Для поля "Сумма" в документе можно задать выражение `[Количество] * [Цена]`. Как только пользователь изменит количество или цену, сумма мгновенно пересчитается.

#### `Только чтение` (`readOnly`)
*   **Что делает:** Запрещает пользователю вручную редактировать значение поля.
*   **Как работает:** Поле становится неактивным (серым). Это свойство также автоматически включается, если для поля задано `Выражение`.

#### `Скрыть` (`hidden`)
*   **Что делает:** Полностью скрывает поле из интерфейса формы.
*   **Как работает:** Поле не отображается на форме, но его значение существует и может использоваться в `Выражениях` или `Автозаполнении`.

#### `Автозаполнение` (`autofillFrom`)
*   **Что делает:** Автоматически заполняет числовое поле значением из другого объекта по ссылке.
*   **Как работает:** Если в форме есть поле-ссылка (например, "Товар"), то для поля "Цена" можно настроить автозаполнение из реквизита "Цена" выбранного товара. Когда пользователь выберет товар, его цена автоматически подставится в соответствующее поле.

---

### Свойства с особым поведением

#### `Уникальное` (`unique`)
*   **Что делает:** Гарантирует, что значение в этом поле не будет повторяться.
*   **Как работает:** Проверка уникальности происходит на уровне **базы данных в момент сохранения**. Если такое число уже есть в другой записи, система выдаст ошибку после нажатия кнопки "Сохранить". Проактивной проверки во время ввода нет.

---

### Не реализованные свойства

Эти свойства можно найти в редакторе метаданных, но, согласно анализу кода, они **не оказывают никакого влияния** на поведение числовых полей.

#### `Точность` (`precision`)
*   **Что делает:** Ограничивает количество знаков после запятой, разрешенных для ввода.
*   **Как работает:** Ограничивает количество знаков после запятой, разрешенных для ввода.

#### `Мин. значение` (`minValue`)
*   **Что делает:** Запрещает ввод чисел, которые меньше указанного минимального значения.
*   **Как работает:** Проверяет на минимальное значение при вводе или сохранении.

#### `Макс. значение` (`maxValue`)
*   **Что делает:** Запрещает ввод чисел, которые больше указанного максимального значения.
*   **Как работает:** Проверяет на максимальное значение при вводе или сохранении.
