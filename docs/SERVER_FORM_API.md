# Серверный formApi для проведения документов

## Обзор

Начиная с версии 0.2.90, серверный API проведения документов (`/api/documents/posting`) поддерживает полный набор методов `formApi` для использования в модулях форм при проведении/отмене проведения документов.

**Версия:** 0.3.0 (добавлены расширенные возможности)

---

## 🆕 Новые возможности (v0.3.0)

### **1. Расширенная фильтрация `readList()`**

Поддерживаются операторы сравнения в именах полей:

| Оператор | Описание | Пример |
|----------|----------|--------|
| `[gt]` | Больше | `{'Сумма[gt]': 1000}` |
| `[lt]` | Меньше | `{'Сумма[lt]': 500}` |
| `[gte]` | Больше или равно | `{'Дата[gte]': '2024-01-01'}` |
| `[lte]` | Меньше или равно | `{'Дата[lte]': '2024-12-31'}` |
| `[ne]` | Не равно | `{'Статус[ne]': 'Отменён'}` |
| `[contains]` | Содержит подстроку | `{'Наименование[contains]': 'ООО'}` |

**Примеры:**

```javascript
async function ПередПроведением(formApi) {
  // Найти заказы на сумму больше 10000
  const крупныеЗаказы = await formApi.readList('Документ', 'Заказы', {
    'Сумма[gt]': 10000
  });
  
  // Найти товары с ценой меньше 100
  const дешёвыеТовары = await formApi.readList('Справочник', 'Товары', {
    'Цена[lte]': 100
  });
  
  // Найти контрагентов с названием containing "Ромашка"
  const ромашки = await formApi.readList('Справочник', 'Контрагенты', {
    'Наименование[contains]': 'Ромашка'
  });
  
  // Комбинированный фильтр
  const активныеЗаказы = await formApi.readList('Документ', 'Заказы', {
    'Статус[ne]': 'Отменён',
    'Сумма[gt]': 0,
    'Дата[gte]': '2024-01-01'
  });
}
```

### **2. Кэширование `read()` с раскрытием ссылок**

Метод `read()` теперь:
- **Кэширует** прочитанные объекты (не читает одно и то же дважды)
- **Автоматически раскрывает** ссылочные реквизиты (1 уровень)

**Пример:**

```javascript
async function ПослеПроведения(formApi) {
  const контекст = formApi.getContext();
  
  // Первое чтение — запрос к БД
  const клиент = await formApi.read(контекст.currentObject.link);
  console.log(клиент.Наименование); // Из БД
  
  // Второе чтение — из кэша (быстрее!)
  const тотЖеКлиент = await formApi.read(контекст.currentObject.link);
  console.log(тотЖеКлиент.Наименование); // Из кэша
  
  // Автоматическое раскрытие ссылок
  const заказ = await formApi.read('doc_orders.uuid-123');
  console.log(заказ.Клиент.Наименование); // Раскрыто автоматически!
  console.log(заказ.Клиент.Email); // Доступно без дополнительного read()
}
```

**Логирование:**
```
[Server read] Read and cached: doc_orders.uuid-123
[Server read] Read and cached: cat_clients.uuid-456
[Server read] Cache hit: doc_orders.uuid-123
```

### **3. Валидация данных при `create/update`**

Методы теперь выполняют базовую валидацию:

**`create()`:**
- ✅ Автогенерация `id` (UUID) если не указан
- ✅ Автогенерация `Код` для справочников (максимальный + 1)
- ✅ Автогенерация `Номер` и `Дата` для документов
- ✅ Проверка наличия метаданных

**`update()`:**
- ✅ Проверка существования объекта
- ✅ Объединение с существующими данными (не полная замена)

**`delete()`:**
- ✅ Проверка существования объекта перед удалением

## Доступные методы

### ✅ Приоритет 1: Критически важные методы

#### `update(link, data)` — Обновление объекта

Обновляет существующий объект по ссылке.

**Параметры:**
- `link: string | { id?: string; _metaId?: string }` — Ссылка в формате "MetaId.ObjectId" или объект с `id` и `_metaId`
- `data: any` — Объект с обновляемыми данными

**Возвращает:** Обновлённый объект

**Пример:**
```javascript
async function ПослеПроведения(formApi) {
  const контекст = formApi.getContext();
  
  // Обновить статус заказа
  await formApi.update(контекст.currentObject.link, {
    'Статус': 'В работе',
    'ДатаПроведения': new Date()
  });
}
```

---

#### `create(type, name, data)` — Создание объекта

Создаёт новый объект указанного типа.

**Параметры:**
- `type: string` — Тип объекта: "Регистр", "Документ", "Справочник"
- `name: string` — Имя объекта из метаданных
- `data: any` — Данные объекта

**Возвращает:** Созданный объект с `id`, `_metaId`, `link`

**Пример:**
```javascript
async function ПослеПроведения(formApi) {
  const контекст = formApi.getContext();
  const номер = formApi.getValue('Номер');
  
  // Создать задачу менеджеру
  await formApi.create('Документ', 'Задачи', {
    'Наименование': `Обработка заказа ${номер}`,
    'Ответственный': контекст.currentUser.id,
    'Срок': new Date(Date.now() + 86400000),
    'Статус': 'Новая'
  });
}
```

**Автогенерация:**
- Для **справочников**: автоматически заполняется `Код` (максимальный + 1)
- Для **документов**: автоматически заполняется `Номер` (максимальный + 1) и `Дата` (текущая)
- Для **всех объектов**: генерируется `id` (UUID) если не указан

---

#### `delete(link)` — Удаление объекта

Удаляет объект по ссылке.

**Параметры:**
- `link: string | { id?: string; _metaId?: string }` — Ссылка в формате "MetaId.ObjectId"

**Возвращает:** `{ success: true, id: string }`

**Пример:**
```javascript
async function ПослеОтменыПроведения(formApi) {
  const контекст = formApi.getContext();
  
  // Удалить временные резервы
  const резервы = await formApi.readList('Регистр', 'ВременныеРезервы', {
    'Заказ': контекст.currentObject?.id
  });
  
  for (const резерв of резервы) {
    await formApi.delete(резерв.link);
  }
}
```

---

#### `Почта(toEmail, htmlBody, subject)` — Отправка email

Отправляет HTML-письмо на указанный адрес через `/api/email`.

**Параметры:**
- `toEmail: string` — Email получателя
- `htmlBody: string` — HTML-тело письма
- `subject: string` (опционально) — Тема письма (по умолчанию: "Системное сообщение")

**Возвращает:** `{ success: true, ...result }`

**Пример:**
```javascript
async function ПослеПроведения(formApi) {
  const клиент = await formApi.getValueAsync('Клиент');
  const номер = formApi.getValue('Номер');
  const сумма = formApi.getValue('Сумма');
  
  await formApi.Почта(
    клиент.Email,
    `<h1>Заказ №${номер} проведён</h1>
     <p>Сумма: <b>${сумма} руб.</b></p>
     <p>Спасибо за заказ!</p>`,
    'Заказ проведён'
  );
}
```

### ✅ Приоритет 2: Методы для работы с табличными частями

#### `addTabularSectionRow(sectionName, rowData)` — Добавить строку в ТЧ

**Параметры:**
- `sectionName: string` — Имя табличной части
- `rowData: any` — Данные строки

**Возвращает:** Новая строка с автоматически сгенерированным `id`

**Пример:**
```javascript
async function ПередПроведением(formApi) {
  // Добавить служебную строку доставки
  await formApi.addTabularSectionRow('Товары', {
    'Номенклатура': { id: 'service-delivery', _metaId: 'cat_products' },
    'Количество': 1,
    'Цена': 500,
    'Сумма': 500
  });
}
```

---

#### `updateTabularSectionRow(sectionName, rowIndex, newRowData)` — Обновить строку ТЧ

**Параметры:**
- `sectionName: string` — Имя табличной части
- `rowIndex: number` — Индекс строки (0-based)
- `newRowData: any` — Новые данные (объединяются с существующими)

**Возвращает:** Обновлённая строка

**Пример:**
```javascript
async function ПередПроведением(formApi) {
  const товары = formApi.getTabularSection('Товары');
  
  // Применить скидку 10% ко всем товарам
  for (let i = 0; i < товары.length; i++) {
    const новаяЦена = товары[i].Цена * 0.9;
    await formApi.updateTabularSectionRow('Товары', i, {
      'Цена': новаяЦена,
      'Сумма': новаяЦена * товары[i].Количество
    });
  }
}
```

---

#### `deleteTabularSectionRow(sectionName, rowIndex)` — Удалить строку ТЧ

**Параметры:**
- `sectionName: string` — Имя табличной части
- `rowIndex: number` — Индекс строки (0-based)

**Возвращает:** `{ success: true }`

**Пример:**
```javascript
async function ПередПроведением(formApi) {
  const товары = formApi.getTabularSection('Товары');
  
  // Удалить строки с нулевым количеством
  for (let i = товары.length - 1; i >= 0; i--) {
    if (товары[i].Количество === 0) {
      await formApi.deleteTabularSectionRow('Товары', i);
    }
  }
}
```

## Полный пример использования

```javascript
// В модуле документа "Заказ покупателя"

async function ПередПроведением(formApi) {
  // Проверка наличия товара
  const товары = formApi.getTabularSection('Товары');
  
  for (const товар of товары) {
    const остатки = await formApi.readList('Регистр', 'Остатки', {
      'Номенклатура': товар.Номенклатура?.id
    });
    
    const текущийОстаток = остатки.reduce((sum, r) => sum + r.Количество, 0);
    
    if (текущийОстаток < товар.Количество) {
      formApi.showAlert(
        `Недостаточно "${товар.Номенклатура.Наименование}". ` +
        `Остаток: ${текущийОстаток}, нужно: ${товар.Количество}`
      );
      return false; // Отменить проведение
    }
  }
  
  return true;
}

async function ПослеПроведения(formApi) {
  const контекст = formApi.getContext();
  const номер = formApi.getValue('Номер');
  const сумма = formApi.getValue('Сумма');
  const клиент = await formApi.getValueAsync('Клиент');
  
  // 1. Отправить email клиенту
  await formApi.Почта(
    клиент.Email,
    `<h1>Заказ №${номер} проведён</h1>
     <p>Сумма: <b>${сумма} руб.</b></p>
     <p>Менеджер свяжется с вами в ближайшее время.</p>`,
    'Заказ проведён'
  );
  
  // 2. Создать задачу менеджеру
  await formApi.create('Документ', 'Задачи', {
    'Наименование': `Обработка заказа ${номер}`,
    'Ответственный': контекст.currentUser.id,
    'Срок': new Date(Date.now() + 86400000), // +1 день
    'Статус': 'Новая',
    'Комментарий': `Заказ на сумму ${сумма} руб.`
  });
  
  // 3. Обновить статус заказа
  await formApi.update(контекст.currentObject.link, {
    'Статус': 'В работе',
    'ДатаПроведения': new Date()
  });
  
  // 4. Удалить временные резервы
  const резервы = await formApi.readList('Регистр', 'ВременныеРезервы', {
    'Заказ': контекст.currentObject?.id
  });
  
  for (const резерв of резервы) {
    await formApi.delete(резерв.link);
  }
  
  console.log(`Заказ ${номер} проведён, задача создана`);
}

async function ПослеОтменыПроведения(formApi) {
  const контекст = formApi.getContext();
  const номер = formApi.getValue('Номер');
  
  // 1. Удалить задачу
  const задачи = await formApi.readList('Документ', 'Задачи', {
    'Заказ': контекст.currentObject?.id
  });
  
  for (const задача of задачи) {
    await formApi.delete(задача.link);
  }
  
  // 2. Обновить статус заказа
  await formApi.update(контекст.currentObject.link, {
    'Статус': 'Отменён'
  });
  
  // 3. Отправить уведомление
  const клиент = await formApi.getValueAsync('Клиент');
  if (клиент?.Email) {
    await formApi.Почта(
      клиент.Email,
      `<h1>Заказ №${номер} распроведён</h1>
       <p>Пожалуйста, свяжитесь с нами для уточнения деталей.</p>`,
      'Заказ распроведён'
    );
  }
  
  console.log(`Заказ ${номер} распроведён`);
}
```

## Логирование

Все операции серверного `formApi` логируются в консоль сервера:

```
[Server update] Updated: doc_orders.uuid-abc-123
[Server create] Created: cat_tasks.uuid-xyz-789 (Документ.Задачи)
[Server delete] Deleted: reg_temp_reserves.uuid-temp-123
[Server Почта] Sent to client@example.com: Заказ проведён
[Server addTabularSectionRow] Added row to Товары
[Server updateTabularSectionRow] Updated row 0 in Товары
[Server deleteTabularSectionRow] Deleted row 2 from Товары
```

## Обработка ошибок

Все методы выбрасывают исключения при ошибках, которые можно перехватить в модуле:

```javascript
async function ПослеПроведения(formApi) {
  try {
    await formApi.Почта(client@example.com, '<h1>Тест</h1>', 'Тест');
  } catch (e) {
    console.error('Не удалось отправить письмо:', e.message);
    formApi.showAlert('Ошибка при отправке уведомления');
  }
  
  try {
    await formApi.create('Документ', 'Задачи', { ... });
  } catch (e) {
    console.error('Не удалось создать задачу:', e.message);
  }
}
```

## Ограничения

1. **Транзакционность:** Все операции `create/update/delete` выполняются в рамках текущей транзакции проведения. При ошибке проведения все изменения будут откачены.

2. **Кэширование:** Кэш `read()` действует только в пределах одного вызова проведения/отмены проведения. При следующем проведении кэш будет пустым.

3. **Раскрытие ссылок:** Автоматическое раскрытие работает только для 1 уровня вложенности. Если ссылочный реквизит содержит внутри другую ссылку, она не раскроется.

4. **Табличные части:** Методы `addTabularSectionRow/updateTabularSectionRow/deleteTabularSectionRow` работают только с данными в памяти. Изменения не сохраняются в БД автоматически — нужно сохранить документ через основной механизм проведения.

5. **Валидация:** Базовая валидация выполняется (проверка обязательных полей, существование объектов), но бизнес-валидация (например, проверка уникальности) не выполняется.
