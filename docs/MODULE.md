# Модуль Формы: Полное Руководство

Этот документ подробно описывает все возможности **Модуля Формы** с готовыми примерами кода.

## 1. Что такое Модуль Формы?

**Модуль Формы** — это JavaScript-код, который добавляет сложную логику к вашим формам (справочникам, документам, регистрам).

### Когда он нужен?

- ✅ Динамическое изменение полей (скрыть/показать, сделать обязательным)
- ✅ Проверка сложных условий перед сохранением
- ✅ Автоматическое заполнение полей
- ✅ Работа с данными других объектов (чтение, создание, обновление)
- ✅ Отправка уведомлений (email, сообщения)
- ✅ Проведение документов и работа с регистрами

## 2. API Формы (formApi)

### 2.1. getValue() — Получить значение поля

**Синхронный метод.** Возвращает значение поля без раскрытия ссылочных реквизитов.

```javascript
// Простое значение
const сумма = formApi.getValue('Сумма');
console.log(сумма); // 1000

// Ссылочное значение (вернет только ссылку)
const клиент = formApi.getValue('Клиент');
console.log(клиент); 
// { link: "cat_clients.uuid-123", id: "uuid-123", _metaId: "cat_clients" }
```

**Когда использовать:**
- Для простых полей (числа, строки, даты, булевы значения)
- Когда нужна только ссылка на объект (без деталей)
- В синхронном коде (хуки `ПриИзменении`, `ПередЗаписью`)

---

### 2.2. getValueAsync() — Получить значение с раскрытием ссылок

**Асинхронный метод.** Загружает полные данные ссылочного объекта.

```javascript
// Получаем полные данные клиента
const полныйКлиент = await formApi.getValueAsync('Клиент');
console.log(полныйКлиент);
// {
//   link: "cat_clients.uuid-123",
//   id: "uuid-123",
//   Наименование: "ООО Ромашка",
//   Email: "info@romashka.ru",
//   Телефон: "+7 (495) 123-45-67",
//   Адрес: "г. Москва, ул. Ленина, д. 1"
// }

// Доступ к вложенным реквизитам
const названиеКлиента = полныйКлиент.Наименование;
const email = полныйКлиент.Email;
```

**Когда использовать:**
- Когда нужны реквизиты связанного объекта
- В асинхронных хуках (`ПередЗаписью`, `ПередПроведением`, `ПослеПроведения`)
- Для отображения подробной информации

**Важно:** Использует кэш. Повторный вызов для того же объекта вернет данные из кэша.

---

### 2.3. setValue() — Установить значение поля

**Синхронный метод.** Устанавливает новое значение в поле формы.

```javascript
// Установить простое значение
formApi.setValue('Сумма', 1500);
formApi.setValue('Комментарий', 'Срочный заказ');
formApi.setValue('Дата', new Date());
formApi.setValue('Выполнено', true);

// Установить ссылку
formApi.setValue('Клиент', {
  link: "cat_clients.uuid-123",
  id: "uuid-123",
  _metaId: "cat_clients",
  Наименование: "ООО Ромашка"
});

// Очистить поле
formApi.setValue('Примечание', null);
```

**Автоматизация:** При вызове `setValue` срабатывают:
- Вычисляемые поля (`expression`)
- Автозаполнение (`autofillFrom`)
- Хук `ПриИзменении_ИмяПоля`

---

### 2.4. setFieldState() — Изменить свойства поля

**Синхронный метод.** Управляет видимостью, обязательностью и доступностью поля.

```javascript
// Сделать поле обязательным и видимым
formApi.setFieldState('ПричинаСогласования', {
  required: true,
  hidden: false,
  readOnly: false
});

// Скрыть поле
formApi.setFieldState('Скидка', {
  hidden: true,
  required: false
});

// Сделать поле доступным только для чтения
formApi.setFieldState('Номер', {
  readOnly: true
});

// Сбросить все ограничения
formApi.setFieldState('Комментарий', {
  required: false,
  hidden: false,
  readOnly: false
});
```

**Параметры:**
- `hidden: boolean` — скрыть/показать поле
- `required: boolean` — сделать обязательным
- `readOnly: boolean` — запретить редактирование

---

### 2.5. showAlert() — Показать сообщение пользователю

**Синхронный метод.** Отображает модальное окно с информацией.

```javascript
// Простое сообщение
formApi.showAlert('Документ успешно сохранен');

// Сообщение с заголовком
formApi.showAlert('Недостаточно товара на складе', {
  title: 'Ошибка',
  variant: 'error' // 'info' | 'success' | 'warning' | 'error'
});

// Предупреждение
formApi.showAlert('Скидка превышает максимально допустимую!', {
  title: 'Внимание',
  variant: 'warning'
});

// Успешное сообщение
formApi.showAlert('Письмо отправлено клиенту', {
  title: 'Готово',
  variant: 'success'
});
```

---

### 2.6. showConfirm() — Задать вопрос пользователю

**Синхронный метод.** Показывает диалог с кнопками "Да"/"Нет".

```javascript
// Простое подтверждение
const подтверждено = await formApi.showConfirm('Вы уверены, что хотите удалить этот документ?');
if (подтверждено) {
  // Пользователь нажал "Да"
  await formApi.delete(ссылкаНаДокумент);
} else {
  // Пользователь нажал "Нет"
  return;
}

// Подтверждение с заголовком
const результат = await formApi.showConfirm(
  'Документ будет проведен. Продолжить?',
  { title: 'Проведение документа' }
);
```

**Возвращает:** `true` (Да) или `false` (Нет)

---

### 2.7. preloader() — Показать индикатор загрузки

**Синхронный метод.** Блокирует форму и показывает прогресс выполнения.

```javascript
// Показать прелоадер
formApi.preloader({ show: true, text: 'Загрузка данных...' });

// Выполнить длительную операцию
const данные = await formApi.readList('Регистр', 'Продажи', { 'Месяц': 'Январь' });

// Скрыть прелоадер
formApi.preloader({ show: false });
```

**Полный пример с обработкой ошибок:**

```javascript
async function СформироватьОтчет(formApi) {
  formApi.preloader({ show: true, text: 'Выполняется расчет...' });
  
  try {
    const данные = await formApi.readList('Регистр', 'Продажи');
    const итог = данные.reduce((sum, item) => sum + item.Сумма, 0);
    formApi.setValue('ИтоговаяСумма', итог);
  } catch (error) {
    formApi.showAlert(`Ошибка: ${error.message}`, { variant: 'error' });
  } finally {
    formApi.preloader({ show: false });
  }
}
```

### 2.8. getContext() — Получить контекст выполнения

**Синхронный метод.** Возвращает информацию о текущем пользователе, метаданных объекта и текущем объекте.

```javascript
const контекст = formApi.getContext();
console.log(контекст);
// {
//   currentUser: { id: "user-123", name: "Иванов И.И.", role: "Менеджер" },
//   objectMeta: { id: "doc_orders", name: "Заказы", type: "document" },
//   currentObject: { id: "uuid-abc-123", link: "doc_orders.uuid-abc-123" }
// }

// Проверка прав доступа
if (контекст.currentUser.role === 'Администратор') {
  // Разрешить удаление
}

// Получение ссылки на текущий объект
const ссылкаНаТекущийОбъект = контекст.currentObject?.link;
console.log('Текущий объект:', ссылкаНаТекущийОбъект);
```

**Возвращаемое значение:**
- `currentUser` — информация о текущем пользователе
- `objectMeta` — метаданные текущего объекта (справочник, документ и т.д.)
- `currentObject` — информация о текущем объекте:
  - `id` — уникальный ID объекта
  - `link` — ссылка в формате "MetaId.ObjectId"
  - `null` — если объект новый и ещё не сохранён

## 3. Работа с Табличными Частями

### 3.1. getTabularSection() — Получить строки ТЧ

**Синхронный метод.** Возвращает массив строк табличной части.

```javascript
// Получить все строки
const товары = formApi.getTabularSection('Товары');
console.log(товары);
// [
//   { Номенклатура: { link: "...", id: "..." }, Количество: 10, Цена: 100 },
//   { Номенклатура: { link: "...", id: "..." }, Количество: 5, Цена: 200 }
// ]

// Обработка строк
товары.forEach(строка => {
  console.log(строка.Номенклатура); // Ссылка
  console.log(строка.Количество);   // Число
});

// Подсчет суммы
const сумма = товары.reduce((итог, строка) => {
  return итог + (строка.Количество * строка.Цена);
}, 0);
```

---

### 3.2. getTabularSectionAsync() — Получить строки ТЧ с раскрытием ссылок

**Асинхронный метод.** Возвращает строки с полными данными ссылочных реквизитов.

```javascript
// Получить строки с раскрытыми данными номенклатуры
const товары = await formApi.getTabularSectionAsync('Товары');
console.log(товары[0]);
// {
//   Номенклатура: {
//     link: "cat_goods.uuid-1",
//     id: "uuid-1",
//     Наименование: "Товар 1",
//     Артикул: "ART-001",
//     Цена: 100
//   },
//   Количество: 10,
//   Цена: 100
// }

// Доступ к реквизитам номенклатуры
товары.forEach(строка => {
  console.log(строка.Номенклатура.Наименование);
  console.log(строка.Номенклатура.Артикул);
});
```

---

### 3.3. addTabularSectionRow() — Добавить строку в ТЧ

**Синхронный метод.** Добавляет новую строку в табличную часть.

```javascript
// Добавить одну строку
formApi.addTabularSectionRow('Товары', {
  Номенклатура: {
    link: "cat_goods.uuid-123",
    id: "uuid-123",
    _metaId: "cat_goods",
    Наименование: "Товар 1"
  },
  Количество: 10,
  Цена: 100,
  Сумма: 1000
});

// Добавить несколько строк
const новыеТовары = [
  { Номенклатура: товар1, Количество: 5, Цена: 200 },
  { Номенклатура: товар2, Количество: 3, Цена: 150 }
];
новыеТовары.forEach(товар => {
  formApi.addTabularSectionRow('Товары', товар);
});
```

---

### 3.4. updateTabularSectionRow() — Обновить строку ТЧ

**Синхронный метод.** Изменяет данные в указанной строке (нумерация с 0).

```javascript
// Обновить количество в первой строке
formApi.updateTabularSectionRow('Товары', 0, {
  Количество: 20,
  Сумма: 2000
});

// Обновить цену во второй строке
formApi.updateTabularSectionRow('Товары', 1, {
  Цена: 250,
  Сумма: 750
});

// Частичное обновление (только указанные поля изменятся)
formApi.updateTabularSectionRow('Товары', 0, {
  Скидка: 10 // Остальные поля останутся без изменений
});
```

---

### 3.5. deleteTabularSectionRow() — Удалить строку ТЧ

**Синхронный метод.** Удаляет строку из табличной части (нумерация с 0).

```javascript
// Удалить первую строку
formApi.deleteTabularSectionRow('Товары', 0);

// Удалить последнюю строку
const товары = formApi.getTabularSection('Товары');
formApi.deleteTabularSectionRow('Товары', товары.length - 1);

// Удалить строки по условию
const товары = formApi.getTabularSection('Товары');
for (let i = товары.length - 1; i >= 0; i--) {
  if (товары[i].Количество === 0) {
    formApi.deleteTabularSectionRow('Товары', i);
  }
}
```

**Важно:** Удалять строки нужно **с конца к началу**, чтобы индексы не сбились.

## 4. CRUD API (Работа с данными системы)

### 4.1. read(link) — Прочитать объект по ссылке

**Асинхронный метод.** Загружает полный объект по ссылке.

```javascript
// Прочитать клиента
const клиент = await formApi.read('cat_clients.uuid-123');
console.log(клиент.Наименование); // "ООО Ромашка"
console.log(клиент.Email);        // "info@romashka.ru"

// Прочитать документ
const заказ = await formApi.read('doc_orders.abc-456');
console.log(заказ.Номер);  // "00001"
console.log(заказ.Сумма);  // 15000

// Использовать в хуке
async function ПриИзменении_Клиент(newValue, formApi) {
  if (!newValue?.link) return;
  
  const клиент = await formApi.read(newValue.link);
  formApi.setValue('АдресДоставки', клиент.Адрес);
  formApi.setValue('Телефон', клиент.Телефон);
}
```

**Кэширование:** Повторный вызов для той же ссылки вернет данные из кэша.

---

### 4.2. readList() — Прочитать список объектов

**Асинхронный метод.** Возвращает массив объектов по фильтру.

```javascript
// Получить все активные договоры
const договоры = await formApi.readList('Документ', 'Договоры', {
  'Статус': 'Активен'
});

// Получить товары конкретного поставщика
const товары = await formApi.readList('Справочник', 'Товары', {
  'Поставщик': поставщикId
});

// Сложный фильтр (AND)
const заказы = await formApi.readList('Документ', 'Заказы', {
  'Клиент': клиентId,
  'Статус': 'В работе',
  'Дата': new Date('2024-01-01')
});

// Пустой фильтр (получить все)
const всеТовары = await formApi.readList('Справочник', 'Товары');
```

**Фильтрация:**
- По обычным полям: `{ 'Поле': 'Значение' }`
- По ссылкам: `{ 'Клиент': ссылкаНаКлиента }`
- По ID: `{ id: 'uuid-123' }`

---

### 4.3. update(link, data) — Обновить объект

**Асинхронный метод.** Обновляет существующий объект по ссылке.

```javascript
// Обновить статус заказа
await formApi.update('doc_orders.uuid-123', {
  'Статус': 'Выполнен',
  'ДатаВыполнения': new Date()
});

// Обновить цену товара
await formApi.update('cat_goods.abc-456', {
  'Цена': 150,
  'ДатаИзмененияЦены': new Date()
});

// Массовое обновление
const заказы = await formApi.readList('Документ', 'Заказы', {
  'Статус': 'Черновик'
});

for (const заказ of заказы) {
  await formApi.update(заказ.link, {
    'Комментарий': 'Архивный заказ'
  });
}
```

**Важно:** Обновляются только указанные поля. Остальные остаются без изменений.

---

### 4.4. create(type, name, data) — Создать объект

**Асинхронный метод.** Создает новый объект указанного типа.

```javascript
// Создать нового клиента
const новыйКлиент = await formApi.create('Справочник', 'Клиенты', {
  'Наименование': 'ООО "Вектор"',
  'ИНН': '7701234567',
  'Email': 'info@vector.ru',
  'Телефон': '+7 (495) 987-65-43'
});

console.log(новыйКлиент.id);   // "uuid-new-123"
console.log(новыйКлиент.link); // "cat_clients.uuid-new-123"

// Создать документ
const новаяНакладная = await formApi.create('Документ', 'РасходнаяНакладная', {
  'Дата': new Date(),
  'Клиент': клиентLink,
  'Сумма': 15000
});

// Создать и сразу использовать
const договор = await formApi.create('Документ', 'Договоры', {
  'Клиент': клиентLink,
  'Дата': new Date(),
  'Статус': 'Активен'
});

formApi.setValue('Договор', договор.link);
```

**Автогенерация:**
- Для справочников: автоматически заполняется `Код`
- Для документов: автоматически заполняется `Номер` и `Дата`

### 4.5. delete(link) — Удалить объект

**Асинхронный метод.** Безвозвратно удаляет объект по ссылке.

```javascript
// Удалить один объект
await formApi.delete('cat_goods.uuid-123');

// Удалить по условию
const дубликаты = await formApi.readList('Справочник', 'Товары', {
  'Артикул': 'TMP-001'
});

// Оставляем первый, удаляем остальные
for (let i = 1; i < дубликаты.length; i++) {
  await formApi.delete(дубликаты[i].link);
}

// Каскадное удаление
async function УдалитьЗаказСРезервами(заказLink, formApi) {
  // Найти все резервы этого заказа
  const резервы = await formApi.readList('Регистр', 'Резервы', {
    'Заказ': заказLink.id
  });
  
  // Удалить резервы
  for (const резерв of резервы) {
    await formApi.delete(резерв.link);
  }
  
  // Удалить заказ
  await formApi.delete(заказLink);
}
```

**Важно:** Удаление необратимо! Перед удалением рекомендуется показать подтверждение.

## 5. Интеграции

### 5.1. Почта() — Отправить email

**Асинхронный метод.** Отправляет HTML-письмо на указанный адрес.

```javascript
// Простое письмо
await formApi.Почта(
  'client@example.com',
  '<h1>Ваш заказ готов!</h1><p>Заберите его в нашем офисе.</p>',
  'Статус заказа изменен'
);

// Письмо с данными из формы
async function ОтправитьУведомление(formApi) {
  const клиент = await formApi.getValueAsync('Клиент');
  const номер = formApi.getValue('Номер');
  const сумма = formApi.getValue('Сумма');
  
  const html = `
    <h1>Уведомление по заказу №${номер}</h1>
    <p>Уважаемый(ая) ${клиент.Наименование},</p>
    <p>Ваш заказ на сумму <b>${сумма} руб.</b> успешно проведен.</p>
    <p>Спасибо за сотрудничество!</p>
    <hr>
    <p style="color: gray;">Это письмо отправлено автоматически.</p>
  `;
  
  await formApi.Почта(клиент.Email, html, `Заказ №${номер} проведен`);
}
```

## 6. События (Хуки) Формы

### 6.1. ПриОткрытии(formApi)

Срабатывает **до** загрузки данных. Для начальной настройки формы.

```javascript
function ПриОткрытии(formApi) {
  // Установить дату по умолчанию
  formApi.setValue('Дата', new Date());
  
  // Скрыть технические поля
  formApi.setFieldState('СлужебнаяИнформация', { hidden: true });
  
  // Получить контекст
  const контекст = formApi.getContext();
  console.log('Пользователь:', контекст.currentUser.name);
}
```

---

### 6.2. ПослеОткрытия(formApi)

Срабатывает **после** загрузки данных. Для логики, зависящей от данных.

```javascript
async function ПослеОткрытия(formApi) {
  // Скрыть поля, если документ проведен
  const проведен = formApi.getValue('Проведен');
  if (проведен) {
    formApi.setFieldState('Комментарий', { readOnly: true });
    formApi.setFieldState('Сумма', { readOnly: true });
  }
  
  // Загрузить дополнительные данные
  const клиентId = formApi.getValue('Клиент')?.id;
  if (клиентId) {
    const скидки = await formApi.readList('Регистр', 'Скидки', {
      'Клиент': клиентId
    });
    if (скидки.length > 0) {

  }

}
```

### 6.10. async ПослеОтменыПроведения(formApi) (только документы)

```javascript
async function ПослеОтменыПроведения(formApi) {
  const номер = formApi.getValue('Номер');
  const контекст = formApi.getContext();

  const задачи = await formApi.readList('Документ', 'Задачи', {
    'Заказ': formApi.getValue('Ссылка').id
  });
  
  for (const задача of задачи) {
```

- Срабатывает **после** того, как все движения по регистрам уже удалены
- Нельзя отменить выполнение (нет возврата значения)
- Рекомендуется для:

  - Отправки уведомлений о распроведении
  - Очистки связанных данных (задачи, резервы)

## 8. Советы и лучшие практики

### ✅ Делайте

- Используйте `getValueAsync()` только когда нужны реквизиты связанного объекта
- Кэшируйте результаты `read()` и `readList()` в переменные
- Показывайте `preloader()` для длительных операций
- Обрабатывайте ошибки через `try/catch`
- Возвращайте `false` из `ПередЗаписью` для отмены сохранения

### ❌ Не делайте

- Не вызывайте `getValueAsync()` в синхронных хуках (`ПриИзменении`)
- Не делайте запросы в цикле без необходимости (используйте `readList`)
- Не забывайте скрывать `preloader()` в `finally`
- Не игнорируйте ошибки

### Оптимизация

```javascript
// ❌ Плохо: запрос в цикле
for (const товар of товары) {
  const номенклатура = await formApi.read(товар.Номенклатура.link);
  // ...
}

// ✅ Хорошо: один запрос readList
const всеНоменклатуры = await formApi.readList('Справочник', 'Товары');
// Найти нужные в массиве
```
