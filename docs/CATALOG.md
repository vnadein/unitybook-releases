# Что такое "Справочник"? (Руководство пользователя)

Представьте себе картотеку в библиотеке или список контактов в телефоне. **Справочник** в системе — это то же самое: централизованное место для хранения однотипной информации, которая используется постоянно.

Например, вместо того чтобы каждый раз вручную вписывать имя клиента в документ, вы просто выбираете его из справочника "Клиенты". Это исключает опечатки и гарантирует порядок в данных.

**Примеры справочников:**
*   **Клиенты:** Список всех ваших покупателей и поставщиков.
*   **Товары и услуги:** Полный перечень того, что вы продаете.
*   **Сотрудники:** База данных всех работников компании.
*   **Склады:** Места хранения ваших товаров.

---

## Из чего состоит Справочник?

Каждый справочник, который вы создаете, имеет несколько ключевых настроек, определяющих его поведение и внешний вид.

### Основные настройки

*   `name` (**Название**): Полное имя, которое все видят. Например, "Клиенты".
*   `abbreviation` (**Аббревиатура**): Короткое имя для компактного отображения. Например, "Клиент.".

### Иерархия (Папки)

*   `isHierarchical` (**Иерархический**): Если включить эту опцию (`true`), в справочнике можно будет создавать папки (группы). Это идеально подходит для каталога товаров, где их можно разложить по категориям (например, "Одежда" -> "Мужская" -> "Рубашки"). Если опция выключена, справочник будет представлять собой простой, плоский список.

### Автоматическая нумерация

*   `autoNumbering` (**Автонумерация**): Если включить эту опцию (`true`), система будет автоматически присваивать уникальный номер (Код) каждому новому элементу. Это избавляет от необходимости придумывать коды вручную.

## Содержимое: Реквизиты и Табличные части

"Начинка" каждого элемента справочника (например, каждой карточки клиента) определяется **реквизитами** и **табличными частями**.

### 1. Реквизиты (`fields`)

**Реквизиты** — это поля, из которых состоит карточка элемента. Например, для клиента это могут быть поля "Название", "Адрес", "Телефон", "ИНН".

В системе есть множество типов реквизитов для разных задач:
*   [**Строка**](./STRING.md): Для любого текста.
*   [**Число**](./NUMBER.md): Для цен, количеств и т.д.
*   [**Булево (Да/Нет)**](./BOOL.md): Для флажков, например, "Активен".
*   [**Дата**](./DATE.md): Для выбора даты из календаря.
*   [**Справочник Ссылка**](./LINK_CATALOG.md): Для связи с другим справочником.
*   [**Документ Ссылка**](./LINK_DOC.md): Для связи с документом.
*   [**Кнопка**](./BUTTON.md), [**Надпись**](./TEXT.md), [**Картинка**](./IMAGE.md): Декоративные элементы для улучшения интерфейса.

> Подробную информацию о каждом типе реквизита и его настройках вы можете найти в соответствующих файлах документации.

### 2. Табличные части (`tabularSections`)

**Табличные части** — это вложенные таблицы, которые можно разместить внутри карточки одного элемента. Они идеально подходят для хранения списков информации, например, состава комплекта или списка контактных лиц.

> Подробное руководство по созданию и настройке табличных частей вы найдете в файле [**TABULAR_SECTIONS.md**](./TABULAR_SECTIONS.md).

## Внешний вид и логика

### Настройка формы (`layout`)

Это свойство позволяет вам визуально конструировать внешний вид формы: располагать поля и таблицы в нужном порядке, объединять их в группы, создавать колонки и вкладки.

> Подробное руководство по работе с конструктором форм находится в файле [**LAYOUT.md**](./LAYOUT.md).

### Настройка списка (`showInJournal`, `defaultSortField`)

Вы также можете настроить, как будет выглядеть общий список элементов справочника:
*   `showInJournal`: У каждого реквизита есть эта опция. Если она включена (`true`), то для этого реквизита в списке будет своя колонка.
*   `defaultSortField`: Позволяет задать поле, по которому список будет отсортирован по умолчанию.

### Продвинутые возможности

*   **Модуль формы (`module`):** Позволяет подключить к форме справочника собственный код для реализации сложной бизнес-логики, которая не покрывается стандартными настройками. (См. [MODULE.md](./MODULE.md))
*   **Печатные формы (`printForms`):** Дает возможность создавать шаблоны для печати. Например, для справочника "Товары" можно сделать печатную форму "Прайс-лист". (См. [PRINT.md](./PRINT.md))
