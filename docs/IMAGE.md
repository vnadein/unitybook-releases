# Реквизит "Картинка": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем возможностям реквизита "Картинка" (тип `image`). Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

Реквизит "Картинка" — это декоративный элемент, который позволяет отображать на форме статичное изображение.

---

### Основные свойства

#### `Источник` (`imageUrl`)
*   **Что делает:** **Это главное свойство.** Оно определяет, какое именно изображение будет показано.
*   **Как работает:** В это свойство необходимо вставить прямую ссылку (URL) на файл изображения. Система отобразит картинку по этой ссылке. Это может быть как ссылка на изображение в интернете, так и на файл, загруженный в саму систему.

---

### Дополнительные свойства

Анализ кода показал, что для "Картинки" есть несколько полезных дополнительных настроек, которые рекомендуется использовать:

#### `Альтернативный текст` (`altText`)
*   **Что делает:** Задает текстовое описание изображения.
*   **Как работает:** Этот текст не виден, если картинка успешно загрузилась. Но он будет показан, если изображение не удастся загрузить, а также используется вспомогательными технологиями (например, для незрячих пользователей). Заполнять это свойство — хорошая практика.

#### `Ширина картинки` (`imageWidth`)
*   **Что делает:** Позволяет задать точную ширину изображения.
*   **Как работает:** Вы можете указать значение в пикселях (например, `'150px'`) или в процентах (`'100%'`).

#### `Высота картинки` (`imageHeight`)
*   **Что делает:** Позволяет задать точную высоту изображения.
*   **Как работает:** Аналогично ширине, можно указать значение в пикселях или процентах. Если указать только одно из измерений (ширину или высоту), второе обычно подстраивается пропорционально.

#### `Скрытый` (`hidden`)
*   **Что делает:** Позволяет скрыть картинку с формы.
*   **Как работает:** Если это свойство установлено в `true`, изображение не будет видно пользователю.

---

### Не реализованные свойства

#### `Автозаполнение` (`autofillFrom`)
*   **Ожидаемое поведение:** Могло бы динамически изменять URL картинки на основе данных из другого поля (например, показывать фото сотрудника при его выборе).
*   **Реальность:** Анализ кода показал, что `Автозаполнение` для реквизита "Картинка" **не реализовано**. Ссылка на изображение (`imageUrl`) является статичной и может быть задана только в метаданных.
