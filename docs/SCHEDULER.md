# Серверный планировщик задач

## Обзор

Серверный планировщик позволяет автоматически запускать код компонентов по расписанию **без необходимости держать UI открытым**. Планировщик работает как фоновый процесс на сервере через API Routes.

## Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                    Next.js API Routes                    │
│  ┌─────────────────────────────────────────────────┐   │
│  │         /api/scheduler (REST API)               │   │
│  └─────────────────────────────────────────────────┘   │
│                            │                             │
│                            ▼                             │
│  ┌─────────────────────────────────────────────────┐   │
│  │      lib/scheduler-service.ts                   │   │
│  │  - Проверка расписания (каждую 1 секунду)       │   │
│  │  - Выполнение задач через vm module             │   │
│  │  - formApi для взаимодействия с данными         │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## Быстрый старт

### 1. Создайте компонент с кодом

```javascript
// Компонент: Синхронизация данных
async function onSchedule(formApi) {
  // Читаем данные из внешнего API
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();
  
  // Создаем записи в справочнике
  for (const item of data) {
    await formApi.create('Справочник', 'Товары', {
      'Наименование': item.name,
      'Артикул': item.sku,
      'Цена': item.price
    });
  }
  
  formApi.showAlert(`Синхронизировано ${data.length} товаров`);
}
```

### 2. Создайте задачу в планировщике

1. Откройте раздел **Планировщик** в конфигураторе
2. Нажмите **Новая задача**
3. Выберите созданный компонент
4. Настройте расписание:
   - **Интервал**: каждые 3600 секунд (1 час)
   - **Ежедневно**: в 02:00
   - **Еженедельно**: по понедельникам в 09:00
   - **Cron**: `0 */6 * * *` (каждые 6 часов)

### 3. Запустите серверный планировщик

В редакторе планировщика нажмите кнопку **"Запустить сервер"**.

Индикатор покажет статус:
- 🟢 **Сервер запущен** - задачи выполняются по расписанию
- ⚫ **Сервер остановлен** - задачи не выполняются

## API планировщика

### GET /api/scheduler

Получить статус планировщика:
```bash
curl http://localhost:3000/api/scheduler?action=status
```

Ответ:
```json
{
  "isRunning": true,
  "taskCount": 5,
  "nextExecution": {
    "taskName": "Синхронизация товаров",
    "time": "2024-01-15T02:00:00.000Z"
  }
}
```

### POST /api/scheduler

Управление планировщиком:
```bash
curl -X POST http://localhost:3000/api/scheduler \
  -H "Content-Type: application/json" \
  -d '{"action": "start", "metadata": {...}}'
```

Действия:
- `start` - Запустить планировщик (опционально с метаданными)
- `stop` - Остановить планировщик
- `reload` - Перезагрузить задачи из метаданных

## Типы расписаний

### Интервал
Выполнение каждые N секунд:
```
Интервал: 300 секунд (5 минут)
```

### Ежедневно
Выполнение каждый день в указанное время:
```
Время: 02:00
```

### Еженедельно
Выполнение каждый неделю в указанный день и время:
```
День: Понедельник
Время: 09:00
```

### Ежемесячно
Выполнение каждый месяц в указанный день и время:
```
День месяца: 1
Время: 00:00
```

### Cron выражения
Гибкая настройка расписания:
```
*/5 * * * *     - Каждые 5 минут
0 */2 * * *     - Каждые 2 часа
0 0 * * 0       - Каждое воскресенье в полночь
0 0 1 * *       - 1-го числа каждого месяца в полночь
```

## Режимы запуска

### Один раз
Задача выполнится 1 раз и будет деактивирована.

### Несколько
Задача выполнится N раз (указывается в поле "Максимальное количество запусков").

### Непрерывно
Задача выполняется бесконечно по расписанию.

## Примеры кода компонентов

### Пример 1: Отправка ежедневного отчета

```javascript
async function onSchedule(formApi) {
  // Получаем заказы за вчера
  const yesterday = new Date();
  yesterday.setDate(yesterday.getDate() - 1);
  
  const orders = await formApi.readList('Документ', 'Заказы', {
    'Дата': { $gte: yesterday.toISOString() }
  });
  
  // Считаем сумму
  const total = orders.reduce((sum, order) => sum + order.Сумма, 0);
  
  // Отправляем отчет (здесь можно подключить почтовый сервис)
  formApi.showAlert(`За вчера: ${orders.length} заказов на сумму ${total}`);
}
```

### Пример 2: Очистка старых данных

```javascript
async function onSchedule(formApi) {
  // Находим документы старше 30 дней
  const cutoffDate = new Date();
  cutoffDate.setDate(cutoffDate.getDate() - 30);
  
  const oldDocuments = await formApi.readList('Документ', 'ВременныеДанные', {
    'Дата': { $lt: cutoffDate.toISOString() }
  });
  
  // Удаляем старые документы
  for (const doc of oldDocuments) {
    await formApi.delete(doc.link);
  }
  
  formApi.showAlert(`Удалено ${oldDocuments.length} старых документов`);
}
```

### Пример 3: Синхронизация с внешним API

```javascript
async function onSchedule(formApi) {
  try {
    const response = await fetch('https://api.currency.com/rates');
    const rates = await response.json();
    
    // Обновляем курсы валют
    for (const rate of rates) {
      const existing = await formApi.readList('Справочник', 'КурсыВалют', {
        'КодВалюты': rate.code
      });
      
      if (existing.length > 0) {
        await formApi.update(existing[0].link, {
          'Курс': rate.value,
          'ДатаОбновления': new Date().toISOString()
        });
      } else {
        await formApi.create('Справочник', 'КурсыВалют', {
          'КодВалюты': rate.code,
          'Наименование': rate.name,
          'Курс': rate.value,
          'ДатаОбновления': new Date().toISOString()
        });
      }
    }
    
    formApi.showAlert('Курсы валют обновлены');
  } catch (error) {
    formApi.showAlert('Ошибка синхронизации: ' + error.message, { variant: 'error' });
  }
}
```

### Тестирование компонента

1. Откройте компонент в редакторе
2. Проверьте код на синтаксические ошибки
3. Создайте задачу с режимом "Один раз"
4. Запустите планировщик кнопкой "Запустить сервер"
5. Подождите выполнения
6. Проверьте консоль сервера

### Частые проблемы

**Задача не выполняется:**
- Проверьте, что планировщик запущен (зеленый индикатор)
- Проверьте, что задача активна (галочка "Активна")
- Проверьте расписание

**Ошибка выполнения кода:**
- Проверьте синтаксис JavaScript
- Убедитесь, что используете async/await для асинхронных операций
- Проверьте консоль сервера на ошибки

**Компонент не найден:**
- Убедитесь, что компонент существует
- Проверьте, что компонент имеет код (JS файл)

**Планировщик не запускается:**
- Проверьте консоль сервера на ошибки
- Убедитесь, что порт 3000 не занят
- Проверьте, что метаданные загружены

## Безопасность

- Код компонентов выполняется в изолированном контексте (vm module)
- Таймаут выполнения: 30 секунд
- Ошибки не прерывают работу планировщика
- При ошибке следующий запуск переносится на 1 минуту

## Производительность

- Проверка задач: каждую 1 секунду
- Рекомендуемый минимальный интервал: 60 секунд
- state хранится в памяти (сбрасывается при рестарте сервера)

## Ограничения

- Состояние хранится в памяти (не сохраняется между рестартами)
- Для production рекомендуется использовать Redis или базу данных
- VM module может быть недоступен в некоторых средах (fallback на Function)

## Интеграция

### Автоматический запуск при старте

Для автоматического запуска планировщика при старте приложения, добавьте в корневой layout или API route:

```typescript
import { startScheduler, setSchedulerMetadata } from '@/lib/scheduler-service';

// При загрузке метаданных
startScheduler();
```

### Мониторинг

Используйте API для мониторинга состояния:

```typescript
// Проверка статуса
const status = await fetch('/api/scheduler?action=status');
const data = await status.json();
console.log(`Running: ${data.isRunning}, Tasks: ${data.taskCount}`);
```
