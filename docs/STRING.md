# Реквизит "Строка": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем свойствам реквизита с типом "Строка". Информация основана на анализе исходного кода проекта, поэтому она отражает реальное поведение системы.

---

### Реализованные свойства

Эти свойства полностью работают. Вы можете смело использовать их при настройке объектов.

#### `Обязательный` (`required`)
*   **Что делает:** Делает поле обязательным для заполнения.
*   **Как работает:**
    1.  К названию поля добавляется красная звездочка (`*`).
    2.  Если поле не заполнено, оно подсвечивается красной рамкой, привлекая внимание.
    3.  Система не позволит сохранить объект, пока поле не будет заполнено.

#### `Длина` (`length`)
*   **Что делает:** Ограничивает максимальное количество символов, которые можно ввести.
*   **Как работает:** Устанавливает стандартный HTML-атрибут `maxLength` для поля ввода. Браузер автоматически не даст ввести больше символов, чем указано.

#### `По умолчанию` (`defaultValue`)
*   **Что делает:** Задает значение, которое будет автоматически подставлено в поле при создании **нового** объекта.
*   **Как работает:** При открытии формы нового (несохраненного) объекта система проверяет это свойство. Если оно задано, его значение появляется в поле. Это не работает для уже существующих, сохраненных объектов.

#### `Многострочный` (`multiline`)
*   **Что делает:** Превращает поле из одной строки в область для ввода большого объема текста.
*   **Как работает:** Вместо `<input>` система отображает `<textarea>`. По умолчанию высота поля составляет 3 строки.

#### `Только чтение` (`readOnly`)
*   **Что делает:** Запрещает пользователю редактировать значение поля.
*   **Как работает:** Поле становится неактивным (серым) и защищенным от изменений. Значение в таком поле может быть установлено только автоматически (например, через `defaultValue` или `autofillFrom`).

#### `Файл` (`isFile`)
*   **Что делает:** Превращает поле в инструмент для прикрепления файлов.
*   **Как работает:**
    *   **Если файл не загружен:** Отображается кнопка для загрузки.
    *   **Если файл загружен:** Появляются кнопки "Просмотреть" (для картинок и PDF), "Скачать" и "Удалить".

#### `Скрыть` (`hidden`)
*   **Что делает:** Полностью скрывает поле и его название из интерфейса формы.
*   **Как работает:** Полю присваивается CSS-класс `hidden`. Несмотря на то что поле не видно, оно все еще существует в данных объекта и может использоваться в расчетах или модулях.

#### `Автозаполнение` (`autofillFrom`)
*   **Что делает:** Автоматически заполняет одно поле на основе значения из другого.
*   **Как работает:** Это свойство настраивается для поля-приемника. В нем указывается поле-источник (которое должно быть типа "Ссылка") и имя реквизита из этого источника. Когда пользователь выбирает значение в поле-источнике, система берет данные из указанного реквизита связанного объекта и подставляет их в поле-приемник.
*   **Пример:** В документе "Счет" есть поле "Контрагент" (ссылка). У поля "Адрес доставки" можно настроить автозаполнение из реквизита "Адрес" выбранного контрагента.

---

### Свойства с особым поведением

#### `Уникальное` (`unique`)
*   **Что делает:** Гарантирует, что значение в этом поле не будет повторяться среди всех элементов данного справочника.
*   **Как работает:** Это свойство работает на уровне **базы данных**. При сохранении объекта, если значение в этом поле уже существует у другой записи, база данных выдаст ошибку, и система не позволит сохранить дубль.
*   **Важно:** Проверка происходит **в момент сохранения**, а не в процессе ввода. Пользователь узнает об ошибке только после нажатия кнопки "Сохранить".

---

#### `Описание` (`description` / `tooltip`)
*   **Что делает:** Появляется иконка с вопросом (?), при наведении на которую отображается текст подсказки.
*   **Как работает:** Текст подсказки можно задать в конфигураторе.

#### `Пароль` (`password`)
*   **Что делает:** Скрывает вводимые символы звездочками (`******`), как в стандартных полях для ввода пароля.
*   **Как работает:** Настройка этого свойства изменяет внешний вид поля.
