# Реквизит "Надпись": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем возможностям реквизита "Надпись" (тип `label`). Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

Реквизит "Надпись" — это декоративный элемент, предназначенный для вывода на форму статического текста, заголовков, пояснений или разделителей. Он не хранит в себе данные и не требует ввода от пользователя.

---

### Основные свойства

#### `Текст` (`text`)
*   **Что делает:** **Это главное свойство.** Оно определяет, какой именно текст будет отображен на форме.
*   **Как работает:**
    1.  Система в первую очередь ищет значение в свойстве `text`.
    2.  Если свойство `text` пустое или отсутствует, система возьмет значение из стандартного свойства `name` (Наименование) реквизита.
*   **Форматирование с помощью HTML:** Вы можете использовать HTML-теги прямо в значении свойства `text` для форматирования. Например:
    *   `<strong>Внимание!</strong>` — отобразит текст **жирным**.
    *   `<em>Важная информация</em>` — отобразит текст *курсивом*.
    *   `Текст с<br>переносом строки` — создаст перенос строки.

---

### Дополнительные свойства

#### Свойства стиля
Хотя в вашем запросе их не было, анализ кода показал, что для надписи можно настраивать стиль. Это делается с помощью следующих свойств:

*   `fontSize`: Устанавливает размер шрифта (например, `'16px'`).
*   `fontWeight`: Устанавливает насыщенность шрифта (например, `'bold'` для жирного).
*   `textAlign`: Задает выравнивание текста (например, `'center'` для выравнивания по центру).

#### `Скрытый` (`hidden`)
*   **Что делает:** Позволяет скрыть надпись.
*   **Как работает:** Если это свойство установлено в `true`, надпись не будет отображаться на форме. Это может быть полезно для динамического отображения информации.

---

### Не реализованные свойства

#### `Источник` (`source`)
*   **Ожидаемое поведение:** Не определено.
*   **Реальность:** Анализ кода показал, что в редакторе метаданных может существовать поле для свойства `source`, но оно **никак не используется** при отображении надписи. Заполнение этого свойства не даст никакого эффекта.
