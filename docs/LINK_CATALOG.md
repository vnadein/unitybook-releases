# Реквизит "Справочник Ссылка": Полное руководство по свойствам

Этот документ — исчерпывающее руководство по всем свойствам реквизита "Справочник Ссылка" (тип `reference`). Информация основана на анализе исходного кода проекта и отражает реальное поведение системы.

Реквизит "Справочник Ссылка" — один из самых важных в системе. Вместо того чтобы просто хранить текст (например, имя клиента), он хранит полноценную ссылку на запись в другом справочнике.

**Как это выглядит:** Поле выглядит как текстовое, но справа от него есть кнопка выбора (`[...]`). При нажатии на нее открывается форма списка, где можно найти и выбрать нужный элемент (например, выбрать клиента из общего списка клиентов).

---

### Реализованные свойства

Эти свойства полностью работают и определяют поведение поля-ссылки.

#### `Ссылка на` (`referenceTo`)
*   **Что делает:** **Это ключевое свойство.** Оно указывает, на какой именно справочник будет ссылаться реквизит.
*   **Как работает:** Вы указываете системный `id` нужного справочника (например, `cat_customers`). Когда пользователь нажмет кнопку выбора, система откроет именно этот справочник.

#### `Обязательный` (`required`)
*   **Что делает:** Делает выбор значения в этом поле обязательным.
*   **Как работает:** К названию поля добавляется красная звездочка (`*`), а пустое поле подсвечивается красной рамкой. Система не позволит сохранить объект, пока в поле не будет выбрано значение.

#### `Только чтение` (`readOnly`)
*   **Что делает:** Запрещает пользователю изменять выбранное значение.
*   **Как работает:** Поле и кнопка выбора становятся неактивными (серыми). Значение в таком поле может быть установлено только автоматически (например, через `По умолчанию` или `Автозаполнение`).

#### `Скрытый` (`hidden`)
*   **Что делает:** Полностью скрывает поле и его название из интерфейса формы.
*   **Как работает:** Поле не отображается, но его значение-ссылка существует и может использоваться в других частях системы.

#### `Автозаполнение` (`autofillFrom`)
*   **Что делает:** Автоматически заполняет другие поля на форме на основе данных из выбранного по ссылке элемента.
*   **Как работает:** Это свойство настраивается **не у самого поля-ссылки**, а у тех полей, которые должны заполниться (например, у текстового поля "Адрес"). В настройках автозаполнения вы указываете поле-ссылку (например, "Контрагент") и выбираете, какой из его реквизитов (например, "Фактический адрес") нужно скопировать. Когда пользователь выберет контрагента, его адрес автоматически подставится в поле "Адрес".

### Фильтрация по родителю (и другим полям)

В системе есть мощный механизм, который позволяет фильтровать список выбора одного поля на основе значения другого. Чаще всего это используется для иерархии "Родитель -> Потомок".

*   **Пользовательский термин:** `Родитель`
*   **Свойство в коде:** `filter`

**Как это работает на примере "Страна -> Город":**

Представьте, у вас есть два поля-ссылки: "Страна" и "Город". Вы хотите, чтобы после выбора страны в списке городов показывались только города из этой страны.

1.  **Настройка:** Для реквизита "Город" в редакторе метаданных настраивается свойство `filter`. В нем указывается, что поле "Страна" в справочнике "Города" должно быть равно значению, выбранному в поле "Страна" на текущей форме.
2.  **Работа в интерфейсе:**
    *   Пользователь открывает форму. Поле "Город" неактивно или показывает все города.
    *   Пользователь выбирает значение в поле "Страна" (например, "Казахстан").
    *   Теперь, когда он нажимает на выбор в поле "Город", в списке он увидит только те города, у которых в их карточке указана страна "Казахстан".

Этот механизм является ключевым для работы с иерархическими и зависимыми данными. Он позволяет создавать очень удобные и логичные интерфейсы, предотвращая выбор неверных данных.
